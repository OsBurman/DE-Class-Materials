## Git Commit Conventions

Format: `type: short description`

| Type | When to use |
|------|-------------|
| `feat` | Adding a new feature |
| `fix` | Fixing a bug |
| `docs` | Documentation only |
| `chore` | Build/tooling changes |
| `refactor` | Code refactoring |
